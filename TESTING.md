# Ollama Integration Testing Guide

Follow these steps to verify that the Ollama integration is functioning correctly within the FastAPI backend.

### Step 1: Verify Model Availability
Run the following command to check if the required model is available:
```bash
ollama list
```
**Expected:** `llama3.2:1b` is present in the list of models.

### Step 2: Verify Ollama Service Status
Check if the Ollama API is responding:
```bash
curl http://localhost:11434/api/tags
```
**Expected:** Ollama responds successfully with a JSON listing of tags/models.

### Step 3: API Health Check
Start the FastAPI server, open the Swagger UI (`http://127.0.0.1:8000/docs`), and visit the health endpoint:
```http
GET /api/v1/llm/health
```
**Expected:** A healthy response indicating successful connection and model availability:
```json
{
  "status": "healthy",
  "ollama_connected": true,
  "model": "llama3.2:1b",
  "model_status": "available"
}
```

### Step 4: Run a Generation Test
Visit the test endpoint:
```http
GET /api/v1/llm/test
```
This endpoint sends the prompt "Explain Artificial Intelligence in 2 lines" to the LLM.
**Expected:** An AI-generated response, along with timing and success metrics:
```json
{
  "success": true,
  "response": "...",
  "response_time_ms": 1234
}
```

### Step 5: Test Failure Modes
Stop the Ollama service on your local machine.

Hit the health endpoint again:
```http
GET /api/v1/llm/health
```
**Expected:** Health endpoint reports unavailable state:
```json
{
  "status": "unhealthy",
  "ollama_connected": false,
  "message": "..."
}
```

### Step 6: Review Application Logs
Check the console and the log file generated by the application.
Navigate to the backend logs directory:
```bash
cat backend/logs/ollama.log
```
**Expected:** Request success, request durations, and connection errors (from Step 5) are properly recorded.
